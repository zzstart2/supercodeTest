SenseNova AI API 文档

SenseNova 提供与 OpenAI 完全兼容的大模型 API 服务。以下文档涵盖所有核心接口及行为规范。

Base URL · 所有接口请求基于 https://api.sensenova.dev/v1

快速开始

使用 SenseNova API 只需 3 步：

  1. 注册账号 并完成手机号验证
  2. 在 控制台 → API Keys 创建一枚 nv- 开头的密钥
  3. 替换 OpenAI SDK 的 base_url 为 SenseNova 地址，即可调用
Python
Node.js
cURL
from openai import OpenAI

client = OpenAI(
    base_url="https://api.sensenova.dev/v1",
    api_key="sn-xxxxxxxxxxxxxxxxxxxx",
)

resp = client.chat.completions.create(
    model="sensenova-chat-pro",
    messages=[{"role": "user", "content": "Hello!"}],
)

print(resp.choices[0].message.content)

鉴权

所有 API 请求必须在 HTTP Header 中携带 Bearer Token：

Authorization: Bearer sn-xxxxxxxxxxxxxxxxxxxx

建议为不同应用/环境创建独立的 API Key，以便独立监控与轮换。密钥可在 控制台 随时注销。

Chat Completions

兼容 OpenAI 协议，支持流式 & 非流式模式。

POST /v1/chat/completions

请求参数

字段类型说明
modelstring模型 ID，如 sensenova-chat-pro
messagesarray对话历史，role ∈ {system, user, assistant}
streambool是否开启 SSE 流式响应（默认 false）
temperaturenumber采样温度，0.0 - 2.0
max_tokensinteger最大生成 token 数
top_pnumbernucleus 采样

响应示例

{
  "id": "chatcmpl-8aBcD...",
  "object": "chat.completion",
  "created": 1713167890,
  "model": "sensenova-chat-pro",
  "choices": [{
    "index": 0,
    "message": {"role": "assistant", "content": "Hello! How can I help?"},
    "finish_reason": "stop"
  }],
  "usage": {
    "prompt_tokens": 12,
    "completion_tokens": 8,
    "total_tokens": 20
  }
}

Models 列表

获取当前账号可用的全部模型与元信息。

GET /v1/models

响应字段包含模型名、上下文长度、支持能力标签 (chat / coding / fim) 等。

可用模型

模型 ID类别上下文能力
sensenova-chat-liteChat32K通用对话（免费档）
sensenova-chat-proChat128K通用对话（Pro 档）
sensenova-code-liteCoding64K代码补全、重构
sensenova-code-proCoding200K仓库级代码理解
sensenova-code-fim补全64K文本补全（prefix + suffix）

文本补全

提供文本补全接口，支持插入 prefix 与 suffix，模型在中间生成。

POST /v1/completions 
{
  "model": "sensenova-code-fim",
  "prompt": "def quick_sort(arr):\n    if len(arr) <= 1:\n        return arr\n    ",
  "suffix": "\n    return quick_sort(left) + [pivot] + quick_sort(right)",
  "max_tokens": 128
}

限流 Rate Limiting

SenseNova 按 API Key 维度执行限流策略：

档位RPMTPM
Free1020,000
Pro60500,000
Team定制，不设硬上限

每次响应都会在 Header 中返回剩余配额：

X-RateLimit-Limit-Requests: 60
X-RateLimit-Remaining-Requests: 42
X-RateLimit-Reset-Requests: 27s
超限行为 · 当 RPM/TPM 超过限额时返回 HTTP 429，带 Retry-After 响应头。SDK 建议实现指数退避。

错误码

HTTP错误码含义
401invalid_api_keyAPI Key 无效或已注销
403model_access_denied当前套餐无权访问该模型
404model_not_found模型 ID 不存在
429rate_limit_exceeded请求速率或 token 速率超限
402insufficient_quota账户余额或额度不足
400content_policy_violation输入内容未通过合规审核
500internal_error服务器内部错误

内容审核

所有请求的输入输出都会经过合规引擎实时扫描，拦截范围包括：

  • 暴力、违法、仇恨类内容
  • 儿童不宜、明显违规场景
  • 越狱 / Prompt 注入尝试（输出侧过滤）

被拦截时返回 400 content_policy_violation，并携带 filter_reason 字段说明触发类别。
