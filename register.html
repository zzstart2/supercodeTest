完善账号信息

手机号已通过校验，下一步设置登录账号与密码。

DEV SPEC · 开发参考

1 进入条件

  • 由 login.html 手机号 Tab 跳转而来，必须携带 ?cc=&phone=&verified=1
  • 未携带 verified=1 或 token 失效 → 返回 login.html 重新校验
  • 同号多账号（verify 返回 ≥ 2）不会进入此页，注册接口对该号码返回 409

2 表单字段

  1. 设置账号：4–32 位，[a-zA-Z0-9_-]，全局唯一，作为登录账号且不可修改
  2. 设置密码：≥ 8 位，需含字母 + 数字；前端展示强度条（弱 / 一般 / 中 / 强）
  3. 确认密码：前端实时一致性提示，提交前需 pwd1 === pwd2
  4. 勾选服务条款 + 隐私政策

3 提交流程

POST /auth/register
入参：{ phone_token, account, password }
phone_token 来自 Step 1 的 /auth/phone/verify，后端再次校验后落库
  • 账号已被占用 → 返回 409，前端字段下提示"该账号已存在"
  • phone_token 失效或对应号码已存在多账号 → 返回 409，前端跳回 login.html
  • 成功 → 自动登录新账号 → 跳 /console/dashboard

4 后续操作

  • 新账号默认绑定该手机号（用于后续"修改密码"重置流程）
  • 注册后用户菜单里"修改密码" → 用此手机号 + 验证码 + 新密码完成重置
  • "更换手机号"链接 → 返回 login.html 重新走 Step 1
我们遵循 SOC2 标准，你的数据始终受到保护。
